# ADR-060: Secure Loopback HTTP Exception for Development

## ステータス

Accepted (2026-03-04)

## コンテキスト

プロジェクトのセキュリティ方針（ADR-019 および ADR-026）に基づき、エンジンリソース（JS/WASM）のロードは HTTPS をデフォルトで強制しています。これは、中間者攻撃（MITM）による悪意あるコードの注入を物理的に遮断するための重要なガードレールです。

しかし、ローカル開発環境や E2E テスト環境においては、`localhost` や `127.0.0.1` などのループバックアドレスを使用して HTTP 経由でリソースを提供することが一般的です。これらを一律に拒否すると、開発体験が著しく低下し、またビルド成果物をローカルで検証する E2E テストが失敗するという課題が発生しました。

W3C の [Secure Contexts](https://w3c.github.io/webappsec-secure-contexts/#is-origin-trustworthy) 仕様および [RFC 6761 §6.3](https://www.rfc-editor.org/rfc/rfc6761#section-6.3) によれば、ループバックアドレス（`localhost`, `127.0.0.1`, `[::1]`）および `*.localhost` サブドメインは「潜在的に信頼できるオリジン」として定義されています。

## 意思決定

セキュリティと開発利便性を両立させるため、以下の例外規定を導入します。

1. **プロトコル検証の緩和**: `EngineLoader` および `SecurityAdvisor` において、URL のプロトコルが `http:` であっても、ホスト名がループバックアドレス（`localhost`, `127.0.0.1`, `[::1]`）または `*.localhost` である場合は、例外（`SECURITY_ERROR`）を投げずに通信を許可します。
2. **判定ロジックの集約**: この「安全なホスト」の判定ロジックは `SecurityAdvisor.isLoopbackHost` に集約し、プロジェクト全体で一貫した基準を適用します。
3. **リモートホストの厳格性維持**: 上記以外のリモートホストに対する `http:` 通信は、引き続き一切許可しません。

## 影響

- **開発体験 (DX)**: ローカルサーバー（`pnpm dev` や Portless）を使用した開発において、特別な設定なしでリソースをロード可能になります。
- **E2E テストの安定化**: ビルド済みのダッシュボードを `localhost` で動かす Playwright テストが、セキュリティエラーで中断されることなくパスするようになります。
- **型安全性と品質**: 修正に伴い `any` を排除し、厳格な型キャストと Lint ルールの遵守を継続します。
- **テストコードの更新**: `EngineLoader.security.test.ts` において、「localhost への HTTP も拒否する」という旧来のテストケースを「許可する」内容に更新しました。
