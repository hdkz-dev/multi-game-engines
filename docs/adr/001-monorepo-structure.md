# ADR-001: モノリポ構成の採用

## ステータス

承認済み

## 日付

2026-01-28

## コンテキスト

プロジェクトは `core` ライブラリと、複数の `adapter-*` パッケージ、およびドキュメントやインフラ設定から構成されます。
これらを個別のリポジトリで管理すると、パッケージ間の依存関係の解決や、一貫したリリース管理が困難になります。

## 決定

npm workspaces を使用したモノリポ（Monorepo）構成を採用します。

### 根拠

1. **コード共有**: `packages/core` の型定義やユーティリティを、各アダプターから容易に参照できます。
2. **アトミックな変更**: インターフェースの変更と、それに対応する各アダプターの修正を一つのコミットで完結させることができます。
3. **ツールの共通化**: ESLint, Prettier, TypeScript の設定をルートで一元管理し、全パッケージで一貫性を保つことができます。

## 結果

- `packages/` 配下に各コンポーネントを配置し、ルートの `package.json` で管理します。
- 開発者はルートディレクトリでの操作により、プロジェクト全体を一度にビルドやテストすることが可能になります。
