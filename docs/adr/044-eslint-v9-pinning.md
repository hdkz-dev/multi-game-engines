# ADR-044: ESLint v9 ピン留めとモノレポ設定の安定化

## ステータス

承認済み

## 日付

2026-02-23

## コンテキスト

ESLint v10.0.0 において、モノレポ構成に影響を与える重大な破壊的変更（設定検索の起点変更）および Node.js 最小要件の引き上げが行われました。
現在のプロジェクト構成および CI 環境において、意図しないビルド失敗を防ぎ、安定した静的解析を提供する必要があります。

## 決定

1. **v9 へのピン留め**: `package.json` において `eslint` を `^9.39.3` に固定し、自動的な v10 へのアップグレードを阻止します。
2. **非同期安全性の強化**: `@typescript-eslint/no-floating-promises` ルールをプロジェクト全体で `error` レベルで有効化します。

## 結果

- モノレポ設定の安定性が確保され、CI での設定読み込みエラーが解消されました。
- Promise の await 漏れが静的に防止されるようになり、非同期処理の信頼性が向上しました。
- 将来的な v10 への移行は、モノレポ全域の設定が対応した段階で再検討します。
