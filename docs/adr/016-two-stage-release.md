# ADR-016: 2段階リリース戦略 (2-Stage Release Roadmap)

- **ステータス**: 承認済み
- **日付**: 2026-02-06

## 1. コンテキスト (Context)

自前で最適化した WASM エンジンのビルドと配信インフラの構築には時間がかかる。一方で、既存の npm パッケージやパブリック CDN 資産を活かせば、早期に実用的な製品を提供できる。開発スピードと最高性能のトレードオフを解消する必要がある。

## 2. 決定 (Decision)

開発を以下の 2 段階（Stage）に分けて進める。

1. **Stage 1: クイックスタート**: 既存の npm 資産（`stockfish` 等）やパブリック CDN（jsDelivr 等）を活用し、アダプター基盤を早期に完成させる。
2. **Stage 2: 究極のパワーと制御**: 自前の Docker ビルドパイプラインを構築し、SIMD128/Threads 最適化を施した独自バイナリを自前 CDN (R2) から供給する。

## 3. 結果 (Consequences)

- **メリット**:
  - プロジェクトの「動くもの」を早期にリリースし、フィードバックを得られる。
  - 将来的にコードを変更せずバイナリだけを高速なものに差し替えられる。
- **デメリット**:
  - Stage 1 ではパブリック CDN の可用性や、既存パッケージのバージョンの制約を受ける。
