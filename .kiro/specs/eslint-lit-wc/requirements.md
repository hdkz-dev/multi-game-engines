# 要件定義書 - ESLint Lit/WC プラグインの導入

## はじめに

`multi-game-engines` プロジェクトにおいて、Lit を使用した Web Components の開発品質を高めるため、`eslint-plugin-lit` および `eslint-plugin-wc` を導入します。これにより、テンプレート内の構文エラーや、Web Components のベストプラクティスに反する実装を自動的に検知・修正可能にします。

## 要件

### 要件 1: Lint プラグインのインストールと基本設定

**目的:** 開発者として、Lit および Web Components 特有のルールを自動チェックできるようにしたい。

#### 受入基準

1. `eslint-plugin-lit` が `devDependencies` に追加されていること。
2. `eslint-plugin-wc` が `devDependencies` に追加されていること。
3. ルートの `eslint.config.mjs` にこれら 2 つのプラグインが適切に設定（Flat Config）されていること。
4. 設定は各プラグインの `recommended` レベルをベースにすること。

### 要件 2: 適用範囲の最適化

**目的:** 不要なパッケージ（純粋なロジックのみのパッケージなど）でのオーバーヘッドを避けつつ、UI 関連パッケージに確実に適用したい。

#### 受入基準

1. `packages/ui-*-elements` などの UI 関連パッケージに対して、プラグインが有効であること。
2. 他のパッケージ（`core` など）で誤検知が発生しないよう、必要に応じてファイルパターンを指定すること（ただし Lit の `html` タグは特徴的なので、全体適用でも問題ないか検討する）。

### 要件 3: 既存コードの検証と修正

**目的:** 導入時に既存のコードをスキャンし、問題があれば修正または例外設定を行う。

#### 受入基準

1. `pnpm lint` を実行し、既存コードにエラーが出ないことを確認する。
2. エラーが出た場合は、プロジェクトの基準に則って修正する。
